# Инструкция: Как собрать мобильное приложение (APK) и почему не работал ИИ

Привет! Если после конвертации проекта в APK-приложение функции ИИ переставали работать, это происходило потому, что мобильному приложению не передавался ваш секретный `GEMINI_API_KEY`, который по умолчанию скрыт в облаке. Я добавил систему "заглушек", чтобы приложение больше не падало с ошибкой, если ключа нет — теперь оно будет работать даже без него, но для того чтобы вернуть *настоящий* ИИ, следуйте этой инструкции.

Вход и регистрация (Firebase Auth) изначально реализованы через Email/Пароль, поэтому они **будут отлично работать в мобильном приложении (APK) без дополнительных настроек**.

---

## 1. Подготовка папки и командной строки

Если вы хотите работать на своём устройстве:
1. В AI Studio нажмите на иконку **Шестеренка / Settings** (в правом верхнем углу) -> **Export to ZIP**.
2. Скачайте архив и распакуйте его в удобную папку на вашем компьютере.
3. Скачайте и установите:
   - **Node.js** (с официального сайта nodejs.org)
   - **Android Studio** (для сборки APK)
4. В распакованной папке откройте терминал (командную строку). В Windows (в папке в проводнике нажмите в строке пути и введите `cmd`, либо откройте VS Code и выберите "Terminal -> New Terminal").

---

## 2. Активация ИИ для мобильного приложения

В корневой папке проекта создайте текстовый файл с названием `.env` (обратите внимание, без имени, только расширение) и добавьте туда ваш ключ:

```env
VITE_GEMINI_API_KEY="ВАШ_КЛЮЧ_ОТ_GOOGLE_GEMINI"
```
*(Когда вы собираете приложение, система Vite возьмёт этот ключ и встроит его в сборку, чтобы приложение могло обращаться к ИИ напрямую с телефона)*

---

## 3. Сборка APK с помощью Capacitor

Теперь выполните следующие команды одну за другой в терминале (командной строке):

1. **Установите зависимости:**
   ```bash
   npm install
   ```

2. **Соберите веб-версию со встроенным API ключом:**
   ```bash
   npm run build
   ```

3. **Создайте и синхронизируйте папку Android:**
   Так как у вас еще нет папки `android` в проекте, её нужно добавить. Выполните:
   ```bash
   npm install @capacitor/android
   npx cap add android
   npx cap sync android
   ```
   *(В будущем, когда папка android уже будет существовать, при изменении кода достаточно будет запускать только `npm run build` и `npx cap sync android`)*

4. **Откройте проект в Android Studio:**
   ```bash
   npx cap open android
   ```

## 4. Финальный шаг в Android Studio

Когда откроется Android Studio:
1. Дождитесь, пока внизу закончится процесс индексации (Sync).
2. В верхнем меню выберите **Build** -> **Build Bundle(s) / APK(s)** -> **Build APK(s)**.
3. Дождитесь завершения сборки. В правом нижнем углу появится сообщение: `APK(s) generated successfully for 1 module.`.
4. Нажмите **Locate** в этом сообщении, и перед вами откроется папка с вашим готовым **app-debug.apk**.

Поздравляю, у вас есть полноценное мобильное приложение с рабочим ИИ и авторизацией!

## 5. Как сделать логотип (иконку) для мобильного приложения?

По умолчанию создание иконки в Capacitor делается через специальный пакет. Подготовьте картинку вашего логотипа и выполните эти простые шаги:

1. **Подготовьте картинки:**
   Создайте свою иконку и экран загрузки (splash screen) в формате PNG без прозрачности:
   - Иконка приложения (например, `icon.png`) — должна быть размером минимум **1024x1024 px**.
   - Экран загрузки (например, `splash.png`) — должен быть размером минимум **2732x2732 px**.

2. **Положите картинки в проект:**
   - Создайте папку `assets` в **корневой папке** вашего всего проекта (там же, где файлы `package.json` и `capacitor.config.json`). **НЕ внутри папки `android`!**
   - Положите туда ваши картинки с точными названиями `icon.png` и `splash.png`.

3. **Сгенерируйте иконки для Android/iOS:**
   Откройте терминал (командную строку) в папке проекта и запустите специальную утилиту:
   ```bash
   npx @capacitor/assets generate
   ```
   *Утилита сама обрежет, создаст закругленные края и распихает все версии иконок нужных размеров по папкам Android проекта.*

4. **Пересоберите приложение:**
   Снова зайдите в Android Studio и скомплируйте проект (`Build -> Build APK(s)`). Теперь ваше приложение будет иметь вашу кастомную иконку и крутой экран начала!

## 6. Как загрузить исходный код мобильного приложения на GitHub?

В архитектуре нашего проекта (Capacitor) **исходным кодом гибридного мобильного приложения является весь проект целиком** — как веб-часть (папка `src`, `package.json`), так и сама "родная" оболочка (папка `android`). Они работают вместе как единое целое. То есть на GitHub заливается вся главная папка!

> ⚠️ **ВАЖНО:** Перед публикацией кода, чтобы ваш секретный ключ от ИИ не оказался в интернете, файл `.env` (который вы создавали в пункте 2) должен быть скрыт. Убедитесь, что в файле `.gitignore` (он лежит в корневой папке) есть строка `.env`.

Вот пошаговый процесс публикации кода:

1. **Создайте репозиторий на GitHub:**
   - Зайдите на свой профиль github.com и нажмите зеленую кнопку **New**.
   - Дайте ему имя (например, `mindup-mobile`).
   - Не ставьте галочки "Add a README file" или "Add .gitignore" — репозиторий должен быть абсолютно пустым. 
   - Нажмите **Create repository**.

2. **Откройте терминал в папке проекта:**
   Откройте терминал (командную строку) **в вашей главной папке**, там где лежит `package.json` и сгенерированная папка `android`.

3. **Свяжите код и отправьте его на GitHub:**
   Скопируйте эти команды и вставляйте в терминал по одной (обязательно замените ссылку в 5-й команде на вашу):

   ```bash
   # Инициализируем систему контроля версий Git
   git init

   # Добавляем все файлы проекта (включая папку android)
   git add .

   # Создаем точку сохранения (коммит)
   git commit -m "Initial commit with Android app"

   # Переключаемся на основную ветку main
   git branch -M main

   # Подключаем ваш удаленный репозиторий (ВСТАВЬТЕ СВОЮ ССЫЛКУ С GITHUB!)
   git remote add origin https://github.com/ВАШ_ЛОГИН/НАЗВАНИЕ_РЕПОЗИРИЯ.git

   # Отправляем всё на сервер GitHub
   git push -u origin main
   ```

**Зачем мы сохраняем папку `android`?** 
Разработчики Capacitor (и Google в том числе) настоятельно рекомендуют сохранять папку платформ (`android` и `ios`) в Git. Это делается для того, чтобы все специальные настройки мобилки — иконки, разрешения (на микрофон/модемы), имя пакета и splash screen — сохранились.

**Если вы внесли изменения в код приложения позже:**
1. Отредактируйте код (в папке `src`).
2. Сохраните изменения для мобильной версии: `npm run build` -> `npx cap sync android`.
3. Отправьте обновления в GitHub: `git add .` -> `git commit -m "обновление"` -> `git push`.
